[[id:7cacbaa3-3995-41cf-8b72-58d6e07468b1][Go]]の宣言構文は、Cと異なっている。Goの設計者は、Cでのわかりにくさを改善するためにGoでの宣言構文を採用したという。

Cのネストした型宣言は順番通りに読めずわかりにくい。

表現しようとしている型の図。

[[file:../images/20231014-pointer.drawio.svg]]

#+caption: Cの宣言構文。ポインタ型で、ポインタが指しているのはintを返す関数型
#+begin_src C
int (*x) () // => * func() int
#+end_src

Goはそのまま左から右に読める。

#+caption: Goの宣言構文。ポインタ型で、ポインタが指しているのはintを返す関数型
#+begin_src go
*func() int
#+end_src
